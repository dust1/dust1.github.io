prepareRefresh - 准备刷新

这是一个受保护的方法，从idea的提示中我们可以看到该方法有子类的实现，具体的实现我们还要去到SpringApplicatoin.class的createApplicationContext方法查看：

我们的是Web项目，所以上下文当然就是ServletWeb类型了。
我们进入到该子类对prepareRefresh的实现中

该方法主要清理Bean工厂的扫描器缓存。实际上的逻辑还是调用父类实现。

那么我们回到父类方法：
首先他要修改上下文的活动状态,并设置启动时间为当前时间的时间戳

然后在上下文环境中初始化占位符资源，这里我猜测是初始化.properties这些配置文件中类似于${name}这样的配置。

然后是对不允许为空的属性的校验。应该类似于数据库配置的username，字段是否为空或者能否正常解析。

initPropertySources 初始化占位符资源

该方法在AbstractApplicationContext下不执行任何操作。这里会有三个实现类。而我们要看的是GenericWebApplicationContext类，因为它是AnnotationConfigServletWebApplicationContext的父类。

通过下载source文档我们可以看到在GenericWebApplicationContext类中该方法的一条注解为：

Replace {@code Servlet}-related property sources.
替换Servlet相关的属性源

这和我一开始想的有点区别啊，我们继续往下追究。

果然，这里传入的是servlet的上下文。而该方法的实现位于StandardServletEnvironment类下

而getPropertySources方法就是在初始化资源后获取的配置文件解析对象。

继续进入该方法

实际上就是检查资源中是否存在以下两个展位字符：

public static final String SERVLET_CONTEXT_PROPERTY_SOURCE_NAME = “servletContextInitParams”;

public static final String SERVLET_CONFIG_PROPERTY_SOURCE_NAME = “servletConfigInitParams”;

若存在则替换.

prepareBeanFactory 准备工厂

这是一个简单的专门用于配置工厂的类：

这里有一个介绍很有意思：

注册早期的后处理器以将内部bean检测为ApplicationListeners。

原文：

Register early post-processor for detecting inner beans as ApplicationListeners.

从这个介绍就能明白Spring后处理的实现方式：通过将其注册为监听器，用于监听Spring初始化的状态并进行调用。而通过查看源码我们可以看到该接口

这个描述就很有意思了，他这里最重要的一点就是这个by this factory.，也就是说Bean后处理器是跟BeanFactory绑定的，并不是试用该上下的所有Bean，其实仔细一想也对，毕竟创建Bean的是工厂，将对Bean创建相关的监听器绑定在工厂上是挺正常的。

而进入AbstractBeanFactory看到接口实现后我们发现，BeanFactory维护着一个后处理器队列，新加入的后处理器会移动到队列末尾，如果先前已经存在了，则将其移除
